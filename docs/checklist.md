# 香港馄饨扫码点餐 Demo - 改造开发清单

## 架构决策
| 决策点 | 结论 | 理由 |
|------|------|------|
| 应用形态 | 复用当前 React SPA | 现有扫码、下单、商家看板链路已具备，改造成本最低 |
| 数据层 | 继续使用 MSW mock | 首版聚焦演示，不引入真实后端与支付集成 |
| 路由策略 | 保留现有 5 条主路由 | 保持演示链路稳定，避免新增页面扩大范围 |
| 支付策略 | 首版“到店付款/能力预留” | 销售已确认先不做本地支付对接 |
| AI 策略 | 轻量推荐/问答（规则或静态 mock） | 满足展示需求，控制实现复杂度 |
| 终端范围 | Web 扫码点餐 + 商家 Web 看板 | 与当前销售沟通结果一致 |

## 执行顺序摘要
- 先修文案质量，再做品类替换。
- 风格升级与支付口径收口要在 AI 展示前完成。
- AI 能力只做轻量层，不改变核心下单链路。

## Definition of Done
- 5 条主路由可访问，核心链路可演示。
- 菜单与品牌语境为香港馄饨，不再是通用川菜数据。
- 视觉风格达到“简洁扁平浅色”要求。
- 支付口径明确为预留/模拟，不误导为已接入本地支付。
- AI 能力可演示推荐与问答，且可一键加购。
- 用户端与商家端状态流转 5 秒内可见同步。

## 非目标提醒
- 不做 GCash 接入。
- 不做自动营销图文/视频生产。
- 不做会员营销系统扩展。
- 不做 AI 行业老板平台能力。
