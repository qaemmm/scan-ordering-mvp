# 点餐系统 MVP — 开发清单

## 架构决策
| 决策点 | 结论 | 理由 |
|--------|------|------|
| 应用形态 | 单仓 React SPA | 用户端和商家端共享类型与 Mock 数据，联调成本最低 |
| 端侧布局 | 用户端 H5 + 商家端 PC | 符合真实使用场景，且不拆双项目 |
| Mock 策略 | MSW | 接口调用形态接近真实后端，后续替换成本低 |
| 状态同步 | 订单详情轮询 3-5 秒 | 演示稳定、实现简单，不引入 WS/SSE |
| UI 组件 | Ant Design Mobile + Ant Design | 交付速度和一致性最佳 |

## 执行顺序与依赖摘要
- 必须先完成 Phase 1 再进入页面开发。
- 页面开发顺序固定：`Entry -> Menu/Cart -> Checkout -> OrderDetail -> Merchant`。
- 状态联动验证必须在商家页和订单详情页都可用后执行。

## 完成定义（Definition of Done）
- 5 页路由全部可访问。
- 用户端可完成下单与状态追踪。
- 商家端可接单并推进状态到完成。
- 用户端在 5 秒内看到商家状态变更。
- 所有 P0 场景在 mock 环境可重复演示。

## 注意事项备忘
- 所有接口先通过 `api/client.ts` 调用，禁止页面中直接写死数据。
- `PICKUP` 只做分支逻辑，不新增页面，防止范围膨胀。
- 保持组件复用，避免页面间复制粘贴。
- 任何架构级决策变动必须写入 `DECISIONS.md`。
